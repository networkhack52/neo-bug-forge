What is ZeroDivisionError?

      

        ZeroDivisionError is a Python built-in exception raised when the second operand
        of a division or modulo operation is zero. It applies to /, //
        (floor division), and % (modulo). Python raises it at runtime — not at
        parse time — so it only appears when the zero value is actually reached.
      

      

        Key insight: The zero is almost never a literal 0 in your source code. It's usually a variable, a database field, or the result of a computation that unexpectedly evaluates to zero at runtime.
      


      
Common Causes


      
1. Empty list passed to an average function

      
The most frequent trigger. len([]) is 0, so dividing by it crashes.

      

        

          ✗ Broken
          
        

        

          
def average(nums):
    return sum(nums) / len(nums)  # ZeroDivisionError if nums is []

        

      

      

        

          ✓ Fixed
          
        

        

          
def average(nums):
    if not nums:
        return 0  # or raise ValueError("Cannot average an empty list")
    return sum(nums) / len(nums)

        

      


      
2. Database / API value that's unexpectedly zero

      
You query a total_sales column and assume it's always non-zero. It isn't.

      

        

          ✗ Broken
          
        

        

          
margin = (profit / total_sales) * 100  # ZeroDivisionError if total_sales == 0

        

      

      

        

          ✓ Fixed
          
        

        

          
margin = (profit / total_sales) * 100 if total_sales else 0

        

      


      
3. Modulo with dynamic denominator

      

        

          ✗ Broken
          
        

        

          
index = item_count % page_size  # ZeroDivisionError if page_size == 0

        

      

      

        

          ✓ Fixed
          
        

        

          
if page_size <= 0:
    raise ValueError(f"page_size must be positive, got {page_size}")
index = item_count % page_size

Frequently Asked Questions


      

        

          Why does Python raise ZeroDivisionError instead of returning infinity?
          +
        

        

          Python integers and floats behave differently. For floats, Python actually does return inf in some cases via IEEE 754 — but for integer division with /, //, or %, it raises ZeroDivisionError. The design decision is that silently producing infinity for integer math would mask more bugs than it solves.
        

      


      

        

          How do I see the full traceback to find which line caused it?
          +
        

        

          Run your script from the terminal (not an IDE run button) and Python will print the full traceback. Look for the line marked ZeroDivisionError: division by zero — the line just above it in the traceback is the exact division operation that failed.
        

      


      

        

          Does this happen in Python 2 and Python 3?
          +
        

        

          Yes, ZeroDivisionError exists in both. However, in Python 2, 5 / 2 performs integer division (returns 2), while Python 3 always returns a float (2.5). The ZeroDivisionError behaviour is identical in both versions.
        

      


      

        

          Can I suppress ZeroDivisionError with numpy?
          +
        

        

          Yes. NumPy divides elementwise and returns nan or inf instead of raising, by default. Use np.errstate(divide='ignore', invalid='ignore') to suppress warnings. For pure Python, use a guard clause instead.
