# Neo Bug Forge — VS Code Extension

> Instantly fix code bugs using Claude AI. Select broken code, press **Cmd+Shift+F**, paste the error — done.

---

## Features

- **Right-click → Fix Selected Code** — select any broken code block, right-click, fix it instantly
- **Cmd+Shift+F** keyboard shortcut — fastest path from error to fix
- **Side panel UI** — paste code + error message, get back:
  - ✅ Fixed code
  - 🧠 Plain-English explanation
  - 📊 Confidence score (0–100%)
  - 🏷 Root cause category
  - 🔀 Unified diff (red/green)
  - 🧪 Unit test that would have caught the bug
- **Apply to Editor** — one click patches your file in place
- **7 language badges** — Python, JS, TS, Rust, Go, Java, C++ (or auto-detect)
- **Secure API key storage** — stored in VS Code's SecretStorage, never in plaintext

## Installation

### From VSIX (manual install)

```bash
# 1. Install dependencies
npm install

# 2. Compile TypeScript
npm run compile

# 3. Package the extension
npm run package
# → produces neo-bug-forge-1.0.0.vsix

# 4. Install in VS Code
code --install-extension neo-bug-forge-1.0.0.vsix
```

### Prerequisites

```bash
npm install -g @vscode/vsce
```

## Setup

1. Open VS Code Command Palette: **Cmd+Shift+P**
2. Run: **Neo Bug Forge: Set API Key**
3. Paste your Anthropic API key (`sk-ant-...`)
4. Key is stored securely — never written to disk in plaintext

Get an API key at: https://console.anthropic.com

## Usage

### Method 1: Context Menu (recommended)
1. Select broken code in any editor
2. Right-click → **Neo Bug Forge: Fix Selected Code**
3. Type or paste the error message in the prompt
4. Panel opens with fix auto-generated

### Method 2: Keyboard Shortcut
1. Select broken code
2. Press **Cmd+Shift+F** (Mac) or **Ctrl+Shift+F** (Windows/Linux)
3. Enter error message → fix appears

### Method 3: Open Panel manually
1. Command Palette → **Neo Bug Forge: Open Panel**
2. Paste code and error message manually
3. Click **⚡ Fix My Bug**

## Configuration

| Setting | Default | Description |
|---|---|---|
| `neo-bug-forge.apiKey` | `""` | Fallback API key (prefer Secret Storage) |
| `neo-bug-forge.model` | `claude-sonnet-4-20250514` | Claude model to use |
| `neo-bug-forge.autoApply` | `false` | Auto-apply fix without confirmation |

## Architecture Notes

The extension follows VS Code's recommended webview pattern:

```
Editor (user action)
      ↓
extension.ts  (registers commands, owns Claude API calls)
      ↓
panel.ts      (WebviewPanel + message bridge)
      ↓
Webview HTML  (self-contained UI, no external deps)
```

Message passing is fully typed. The Claude API call lives entirely in the extension host (Node.js context), never in the webview — keeping your API key safe.
